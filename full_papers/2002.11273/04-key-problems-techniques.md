## 4.1 Overview

The typical workflow of SpGEMM is presented in Figure 4, which has five stages, including size prediction, memory allocation, work partition and load balance, numeric multiplication, and result accumulation. Firstly, the size prediction stage aims to predict the memory footprint of result matrix before real execution. Secondly, the memory allocation stage allocates memory space for result matrix on target device. Thirdly, the objective of the work partition and load balance stage is to design an efficient algorithm to fully exploit the performance of parallel processors. Fourthly, numeric multiplications and partial additions are performed, and a large number of intermediate results are also generated in this stage. Result accumulation desires to reduce these results and calculate final results.
