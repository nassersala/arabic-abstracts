### English Version (Summary)

In 2009, Gentry described the first plausible construction of a fully homomorphic cryptosystem that supports both addition and multiplication (Gentry, 2009). This was a major breakthrough in cryptography.

**Gentry's Approach:**
1. Constructs a **somewhat homomorphic** scheme that supports evaluating low-degree polynomials on encrypted data
2. **Squashes** the decryption procedure to express it as a low-degree polynomial
3. Applies **bootstrapping transformation** to obtain a fully homomorphic scheme

The key insight is to evaluate high-degree polynomials using a decryption procedure expressed as a low-degree polynomial. When the degree of evaluable polynomials exceeds the decryption polynomial degree by factor of two, the scheme becomes **bootstrappable**.

**Key Developments:**

- **Gentry's Construction (2009):** Based on ideal lattices, using GGH-type schemes. Security reduced to worst-case hardness of lattice problems. Uses sparse subset-sum problem (SSSP) for squashing.

- **Ring Learning With Errors (RLWE):** Lyubashevsky et al. (2010) presented RLWE assumption, the ring counterpart of Regev's learning with errors. Enables efficient constructions with worst-case to average-case reduction.

- **Brakerski-Vaikuntanathan (2011):** Constructed somewhat homomorphic encryption based on RLWE, achieving both functional powerfulness and efficiency. Introduced **key dependent message security (KDM security/circular security)** - can securely encrypt polynomial functions of its own secret key. Solved open problem of circular secure somewhat homomorphic encryption.

- **Smart-Vercauteren (2010):** Fully homomorphic scheme with smaller key and ciphertext sizes. Uses arithmetic of cyclotomic number fields. Field generated by polynomial $F(X) = X^{2^k} + 1$. Problem: slow key generation.

- **Gentry-Halevi (2011):** Improved key generation using Discrete Fourier Transform (DFT). Fast key generation but tailored to two-power roots of unity.

- **Stehle-Steinfeld (2010):** Improved Gentry's scheme to $O(n^{3.5})$ bits complexity per elementary binary gate. More aggressive security analysis of SSSP. Relaxed definition to allow negligible decryption error probability.

- **Chunsheng (2012):** Modified Smart-Vercauteren using self-loop bootstrappable technique. Security based only on polynomial coset problem without sparse subset problem assumption.

- **Boneh-Freeman (2011):** Linearly homomorphic signature scheme authenticating vector subspaces. First unforgeable linearly homomorphic signature over $F_2$ (binary fields). Based on finding short vectors in integer lattices. Introduced k-small integer solutions (k-SIS) problem.

**Technical Challenges:**
- Complex key generation with appropriate distributions
- Bootstrapping increases public key size linearly with circuit depth
- Trade-offs between efficiency and security assumptions
- Ciphertext blow-up during repeated operations

**Mathematical Foundations:**
- Ideal lattices
- Sparse subset-sum problem (SSSP)
- Bounded distance decoding (BDD)
- Shortest vector problem (SVP)
- Ring learning with errors (RLWE)
- Cyclotomic number fields

### النسخة العربية

في عام 2009، وصف Gentry أول بناء معقول لنظام تشفيري متماثل كامل يدعم كلاً من الجمع والضرب (Gentry, 2009). كان هذا اختراقاً كبيراً في علم التشفير.

**نهج Gentry:**
1. يبني مخطط **متماثل إلى حد ما** يدعم تقييم متعددات الحدود منخفضة الدرجة على البيانات المشفرة
2. **يضغط** إجراء فك التشفير بحيث يمكن التعبير عنه كمتعددة حدود منخفضة الدرجة
3. يطبق **تحويل التمهيد الذاتي** للحصول على مخطط متماثل كامل

الفكرة الأساسية هي تقييم متعددات حدود عالية الدرجة باستخدام إجراء فك تشفير يُعبر عنه كمتعددة حدود منخفضة الدرجة. عندما تتجاوز درجة متعددات الحدود القابلة للتقييم درجة متعددة حدود فك التشفير بعامل اثنين، يصبح المخطط **قابلاً للتمهيد الذاتي**.

**التطورات الرئيسية:**

**بناء Gentry (2009):** مبني على الشبكات البلورية المثالية، باستخدام مخططات من نوع GGH. تم اختزال الأمان إلى الصلابة في أسوأ الحالات لمسائل الشبكات البلورية. يستخدم مسألة مجموع المجموعة الفرعية المتفرقة (SSSP) للضغط.

**التعلم الحلقي مع الأخطاء (RLWE):** قدم Lyubashevsky وآخرون (2010) افتراض RLWE، النظير الحلقي للتعلم مع الأخطاء لـ Regev. يمكّن من بناءات فعالة مع اختزال من أسوأ حالة إلى متوسط حالة.

**Brakerski-Vaikuntanathan (2011):** بنوا تشفيراً متماثلاً إلى حد ما مبنياً على RLWE، محققين كلاً من القوة الوظيفية والكفاءة. قدموا **أمان الرسالة المعتمد على المفتاح (أمان KDM/الأمان الدائري)** - يمكن تشفير دوال متعددة حدود لمفتاحه السري بشكل آمن. حلوا مشكلة مفتوحة للتشفير المتماثل إلى حد ما الآمن دائرياً.

**Smart-Vercauteren (2010):** مخطط متماثل كامل بأحجام مفاتيح ونصوص مشفرة أصغر. يستخدم حساب الحقول العددية الحلقية. الحقل المولد بواسطة متعددة الحدود $F(X) = X^{2^k} + 1$. المشكلة: توليد مفتاح بطيء.

**Gentry-Halevi (2011):** حسّنوا توليد المفتاح باستخدام تحويل فورييه المنفصل (DFT). توليد مفتاح سريع ولكنه مخصص لجذور الوحدة ذات القوى الثنائية.

**Stehle-Steinfeld (2010):** حسّنوا مخطط Gentry إلى تعقيد $O(n^{3.5})$ بتة لكل بوابة ثنائية أولية. تحليل أمان أكثر عدوانية لـ SSSP. أرخوا التعريف للسماح باحتمال خطأ فك تشفير مهمل.

**Chunsheng (2012):** عدّل Smart-Vercauteren باستخدام تقنية التمهيد الذاتي ذات الحلقة الذاتية. الأمان مبني فقط على مسألة المجموعة المشتركة متعددة الحدود دون افتراض مسألة المجموعة الفرعية المتفرقة.

**Boneh-Freeman (2011):** مخطط توقيع متماثل خطياً يصادق على فضاءات فرعية متجهة. أول مخطط توقيع متماثل خطياً غير قابل للتزوير على $F_2$ (الحقول الثنائية). مبني على إيجاد متجهات قصيرة في شبكات بلورية صحيحة. قدموا مسألة حلول الأعداد الصحيحة الصغيرة k (k-SIS).

**التحديات التقنية:**
- توليد مفتاح معقد بتوزيعات مناسبة
- يزيد التمهيد الذاتي حجم المفتاح العام خطياً مع عمق الدارة
- مفاضلات بين الكفاءة وافتراضات الأمان
- تضخم النص المشفر أثناء العمليات المتكررة

**الأسس الرياضية:**
- الشبكات البلورية المثالية (ideal lattices)
- مسألة مجموع المجموعة الفرعية المتفرقة (SSSP)
- فك الترميز ذو المسافة المحدودة (BDD)
- مسألة أقصر متجه (SVP)
- التعلم الحلقي مع الأخطاء (RLWE)
- الحقول العددية الحلقية (cyclotomic number fields)

**الأهمية:**
اختراق Gentry في عام 2009 حل مشكلة مفتوحة منذ فترة طويلة في علم التشفير وفتح مجالاً جديداً بالكامل للبحث. التطورات اللاحقة حسّنت الكفاءة والأمان وقابلية التطبيق العملي. التشفير المتماثل الكامل يمكّن من:
- الحوسبة السحابية الآمنة على البيانات المشفرة
- معالجة البيانات الحساسة دون فك تشفيرها
- حفظ الخصوصية في تحليلات البيانات الضخمة
- تطبيقات blockchain والعملات المشفرة

على الرغم من التحديات العملية (الأداء، أحجام المفاتيح، تضخم النصوص المشفرة)، يظل التشفير المتماثل الكامل أحد أقوى البدائيات التشفيرية لحماية الخصوصية والحوسبة الآمنة.
