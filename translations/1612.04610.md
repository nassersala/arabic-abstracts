# A Lazy Language Needs a Lazy Type System: Introducing Polymorphic Contexts

**Year:** 2016

### English Abstract
The authors contend that "most type systems that support polymorphic functions are based on a version of System-F" and propose an extension addressing limitations in lazy languages. They demonstrate through examples how current type systems may require unnatural program structures in lazily evaluated contexts. Their key contribution involves enabling "polymorphic functions which returns that type," specifically showing how to implement the strict-state thread monad (ST) using returned existential types. The extension, termed "polymorphic contexts," fundamentally "reverses the relationship between the context of a function call and the called function with respect to where it is decided with which type to instantiate a type variable."
