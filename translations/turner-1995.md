### English Abstract
Functional programming is a good idea, but we haven't got it quite right yet. What we have been doing up to now is weak (or partial) functional programming. What we should be doing is strong (or total) functional programming - in which all computations terminate. This paper proposes an elementary discipline of strong functional programming, in which all computations terminate, and introduces a type distinction between data, known to be finite, and codata, which is potentially infinite.

### Notes
This paper advocates for "strong" or "total" functional programming where all functions are guaranteed to terminate. Turner introduces a type system that distinguishes between finite data and potentially infinite codata (coinductive types). This work influenced the development of total functional languages and dependently-typed systems. Note: The user's table indicated year 2004, but this paper was actually published in 1995. Turner wrote a different paper "Total Functional Programming" in 2004.
