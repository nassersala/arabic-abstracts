# Functional Reasoning for Distributed Systems with Failures

### English Abstract
Distributed system theory literature often argues for correctness using an informal, Hoare-like style of reasoning. We present a formal framework that bridges this style of reasoning with rigorous proof techniques. Our approach features twin languages - Sync and Async - where Sync represents systems as synchronous data-parallel programs with functional denotational semantics suitable for Hoare-style formal reasoning, while Async represents asynchronous node interactions as monadic programs with trace-based operational semantics. We prove that safety properties verified in Sync's denotational semantics are preserved in Async's operational semantics after compilation. We implemented both languages in Rocq and validated the approach using fault-tolerant consensus protocols including BOSCO and SeqPaxos. This work addresses the gap between informal reasoning common in distributed systems literature and formal proof approaches.
