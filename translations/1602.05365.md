# A Specification of Open Transactional Memory for Haskell
## مواصفة ذاكرة المعاملات المفتوحة لـ Haskell

**arXiv ID:** 1602.05365
**Authors:** Marino Miculan, Marco Peressotti
**Year:** 2016
**Categories:** cs.PL
**Translation Quality:** 0.87
**Glossary Terms Used:** concurrent, composable, specification

### English Abstract
The authors present Open Transactional Memory (OTM), a programming model for concurrent systems that enables interactions between composable memory transactions. Rather than enforcing complete isolation, this approach "relaxes isolation between transactions, still ensuring atomicity" by allowing threads from different transactions to access shared variables and then merge together transparently. The model is designed for loosely-coupled interactions driven solely by data access patterns. The researchers provide a formal specification within Concurrent Haskell, demonstrating it extends the existing STM abstraction while maintaining the opacity property, a critical correctness criterion for transactional systems.
